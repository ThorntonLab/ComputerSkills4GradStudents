Reproducing your venv¶

You have created a venv. During the course of your work, you have pip installed a lot of things. What is in your venv?

The pip freeze command¶

Type this (in your activated venv, of course):

pip freeze

Concrete example:

  • If you have an activated venv, deactivate it.

  • Create a new one.

  • Activate it, then

    pip install pandas

  • Now, say

    pip freeze

Saving the results¶

Use redirection to save the output from pip freeze:

pip freeze > lockfile.txt

This “lock” file has the same type of contents as found in requirements.txt. The differences are:

  • The lock file contains nearly everything in your venv. (The “nearly” is because a few packages that are always there are not included.)

  • The precise version of everything is included.

Using your lockfile to recreate your venv elsewhere.¶

You are happy with your venv on your laptop. Your code works, etc.. Now, you need to “scale up” and run your work flow on a more powerful machine:

  • Create a venv on that machine.

  • Activate it.

  • Say:

    pip install -r lockfile.txt

Voila! You have reproduced your environment on another machine.

This file is called a “lock” file because it locks each listed dependency to a specific version.

Another method.¶

Simply install from your requirements.txt. Often, the full “lock” file is overkill.
